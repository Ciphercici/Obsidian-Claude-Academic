# memory 技能

## 触发时机

**自动触发**：每轮对话结束时（用户未发送新消息前）

## 核心工作流

### 步骤 1：检查待记录内容

判断本轮是否有需要记录的内容：
- 是否学习了新知识？
- 是否有未完成的工作？
- 是否有需要延续到下一轮的待办事项？

如果以上都没有，可以跳过记录。

### 步骤 2：生成对话纪要

从本轮对话中提取：

**已完成的**：
- 学会/理解了哪些新知识
- 完成了哪些操作

**待续的**：
- 还有什么没做完
- 需要用户确认的事项
- 下一步要做什么

### 步骤 3：保存到 memory/

文件命名格式：`memory/{YYYY-MM-DD}-{主题slug}.md`

主题Slug 取自当前工作的核心主题，如：
- `ingest-claude-code` — 摄取Claude Code资料
- `lint-wiki-health` — Wiki健康检查
- `query-agent-concept` — 查询Agent概念

### 步骤 4：写入文件

```markdown
---
title: "对话纪要: {主题}"
date: YYYY-MM-DD
tags: [对话纪要]
---

## 本轮完成
- [已完成项 1]
- [已完成项 2]

## 待续事项
- [待续项 1]
- [待续项 2]

## 关联文件
- [[相关wiki页面]]
- raw/xxx/xxx.md
```

## 注意事项

- 只记录有实际价值的内容，避免废话
- 待续事项要具体明确，方便下一轮直接接手
- 如果对话很简短（只是简单问答），可以跳过记录
- memory/ 目录需要存在，如不存在则创建
