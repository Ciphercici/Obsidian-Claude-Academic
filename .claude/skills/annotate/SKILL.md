# annotate 技能

## 触发时机

**用户触发**：
- `/annotate <页面> <标签>` — 给页面添加标签
- "给xxx页面添加批注说xxx"
- "标注这个观点有冲突"

**自动触发**：
- ingest 时发现新旧知识冲突
- 发现有争议的观点

## 标注方式

### 方式 1：标签（快速标记）

在页面 frontmatter 的 `tags` 数组中添加：

```yaml
tags: [待验证, 有争议, 已过时, 新见解, 已确认]
```

| 标签 | 含义 |
|------|------|
| 待验证 | 内容未经确认，需核实 |
| 有争议 | 存在不同观点，需标注冲突 |
| 已过时 | 内容已过时，需更新 |
| 新见解 | 新发现或更新观点 |
| 已确认 | 已被验证的内容 |

### 方式 2：批注（深入说明）

在页面正文中使用 callout 格式：

```markdown
> [!note] 标注内容
> 可以包含 [[双链]] 引用

> [!warning] 警告/冲突标注
> 这个观点与 [[另一个页面]] 的说法有冲突

> [!info] 补充说明
> 相关上下文或背景信息
```

| Callout 类型 | 用途 |
|--------------|------|
| [!note] | 一般备注 |
| [!warning] | 警告/冲突 |
| [!info] | 补充信息 |
| [!tip] | 技巧/建议 |
| [!question] | 待澄清问题 |

## 冲突标注模板

当发现知识冲突时，在页面中添加：

```markdown
## 知识冲突

### 观点A（本文）
[原观点描述]

### 观点B（来自 [[其他页面]]）
[冲突观点描述]

### 对比分析
| 方面 | 观点A | 观点B |
|------|------|------|
| 来源 | xxx | xxx |
| 时间 | xxx | xxx |
```

## 操作示例

### 示例 1：添加标签
```
/annotate ClaudeCode 待验证
```
→ 在 frontmatter 添加 `tags: [待验证]`

### 示例 2：添加冲突批注
```
/annotate ClaudeCode 与ANTHROPIC_API有冲突
```
→ 添加批注：
```markdown
> [!warning] 与 [[ANTHROPIC_API]] 的说法有冲突
> [具体冲突内容]
```

### 示例 3：创建冲突页面
在两个相关页面都添加了批注后，需要选择一个页面创建"知识冲突"区块详细说明。
